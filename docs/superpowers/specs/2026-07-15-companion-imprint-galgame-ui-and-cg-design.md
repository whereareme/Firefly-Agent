# 同行印记 Galgame 界面与章节 CG 设计

## 目标

将同行印记从工具式 Tk 面板改为薄荷绿、简洁的 Galgame 体验，同时保留现有剧情推进、关系状态和主程序直连模型的架构。新增可管理的开始界面背景、剧情配图图鉴，以及每个大章完成后生成的一张章节专属 CG。

## 已确认范围

- 开始界面使用铺满窗口的流萤背景图和左侧标题菜单。
- 左侧遮罩保持半透明，背景延伸到菜单下方；选中项使用浅薄荷绿。
- 左上角显示同行印记连接状态。
- 四张现有流萤图片作为内置背景，默认参与轮换。
- 用户可添加本地 JPG、PNG、WebP，并选择哪些图片参与轮换。
- 只选择一张时固定显示；选择两张及以上时自动轮换。
- 设置中提供剧情 CG 图鉴，并区分“剧情配图”和“章节专属 CG”。
- 图鉴按大章分类，只显示已经在剧情中实际出现或成功生成的图片；未解锁内容完全不显示。
- 每个大章完成后，在图片生成模型可用时生成一张章节专属 CG。
- 生成期间显示全屏 Galgame 过场：“专属 CG 生成中，请「称呼」耐心等待”。
- 称呼优先使用流萤已经确认给予用户的专属称号；不存在时使用“开拓者”。
- 生成成功后立即展示大图，并自动收录到对应章节的“章节专属 CG”。

## 不在本次范围

- 不改变现有关系数值、章节门槛或剧情文本生成规则。
- 不让未完成章节或未出现图片以锁定占位形式泄露。
- 不为同一章节自动生成多张候选图。
- 不增加新的图片供应商或图片编辑器。
- 不把供应商密钥写入 Sidecar 配置或磁盘。

## 界面设计

### 开始界面

窗口背景使用当前选中的图库图片全屏裁切显示。左侧为约 54% 不透明度的浅色遮罩，包含标题与主菜单；菜单选中项采用浅薄荷绿底、深绿色文字和圆点。连接状态固定在左上角，不随背景切换移动。

背景每 8 秒切换一次，使用短淡入淡出。只有一张启用图片时不创建轮换计时器。窗口缩放时保持 `cover` 效果，不拉伸图片，不改变菜单尺寸。

### 背景图库

设置页统一展示内置背景和用户背景：

- 内置背景可参与或退出轮换，但不能删除。
- 用户背景导入后复制到 Sidecar 数据目录，可参与、退出轮换或删除。
- 始终至少保留一张启用图片。
- 删除用户图片时同时删除 Sidecar 数据目录中的副本。

### CG 图鉴

图鉴顶部使用两个分段选项：

- `剧情配图`：剧情过程中实际展示过的配图。
- `章节专属 CG`：大章完成后成功生成的纪念图，每章最多一张。

左侧只列出已有图片的章节，右侧显示当前章节缩略图。点击缩略图进入沉浸式大图查看。图片条目显示标题、所属章节和首次出现或生成时间。尚未出现的章节、图片数量和标题均不展示。

### 章节完成生成过场

当大章结局写入成功后，若主程序已同步“图片模型可用”状态：

1. Sidecar 创建该章节唯一的待生成任务。
2. 剧情窗口切换为全屏生成过场，显示章节完成状态、加载动画和等待文案。
3. 文案称呼取已确认的专属称号，否则使用“开拓者”。
4. 主程序异步执行图片生成，剧情 UI 保持响应。
5. 成功后切换到生成结果大图，并写入章节专属 CG 图鉴。

若图片模型不可用，不创建任务、不显示生成过场，章节按原流程正常结束。

## 数据与文件

Sidecar 数据目录新增：

```text
data/
  visual-library.json
  visuals/
    backgrounds/
    chapter-cg/
```

`visual-library.json` 只保存必要元数据：背景 ID、文件名、来源、是否参与轮换，以及 CG 的章节 ID、标题、生成时间和相对文件名。所有路径保存为相对数据目录的路径，并在读取时进行越界校验。

内置背景继续位于包内资源目录，不复制到数据目录。用户图片和生成 CG 使用原子写入；元数据只在图片落盘成功后更新。

章节进度增加可选的 `user_title`，用于保存流萤在剧情中明确确认的专属称号。剧情模型的结构化事件结果增加可选称号字段；只有本轮流萤台词明确给出同一个称号时才接受并持久化，空值或无法在台词中核对的值一律忽略。旧进度读取时默认为空，不影响现有存档。界面称呼解析规则固定为：

```text
trimmed user_title or "开拓者"
```

## 图片生成桥接

图片生成继续由 Firefly 主程序执行，复用现有 `run_direct_image_generation()`、当前图片模型配置和流萤脸部参考技能。Sidecar 不保存 API Key。

Sidecar 提供仅限回环地址访问的最小控制接口：

- 查询待生成的章节 CG 任务。
- 回报任务成功或失败。
- 同步当前图片模型是否可用。

主程序的 `CompanionImprintController` 使用现有后台任务模式轮询待办。收到任务后，将该章已归档剧情摘要、关键事件、关系变化和章节标题组合成提示词，并指定 Sidecar 数据目录下的目标文件。生成任务沿用现有 10 分钟图片超时，不阻塞聊天线程和 Qt 主线程。

同一章节任务使用稳定章节 ID 去重。程序重启后，未完成任务仍可继续；已存在成功 CG 的章节不会再次自动生成。

## CG 提示词

提示词只总结已发生内容，不添加未来剧情。流萤图像必须复用项目内置的 `firefly-face-reference` 参考图和脸眼规则。场景、动作、服装和情绪来自该章归档，不额外加入用户没有要求的暴露、服装或关系设定。

## 失败处理

- 图片模型不可用：跳过生成，正常完成章节。
- 生成失败或超时：保留失败状态，不写入图鉴；生成过场显示简短失败提示并允许返回开始界面。
- 进程退出：待生成任务保留，下一次主程序和 Sidecar 同时在线时重试。
- 输出文件损坏或不存在：不写入成功记录，并允许重试。
- 背景文件被外部删除：忽略失效条目并回退到第一张可用内置背景。
- 图库元数据损坏：保留损坏文件副本，使用四张内置背景启动，不影响剧情存档。

## 代码边界

- `relationship_gateway/narrative.py`：章节完成状态和可选专属称号，不负责图片 IO。
- `relationship_gateway/visual_assets.py`：扩展为背景与 CG 元数据、路径校验和原子持久化。
- `relationship_gateway/gateway.py`：仅增加回环控制端点和章节 CG 任务状态。
- `relationship_gateway/panel.py`：开始界面、背景设置、图鉴和生成过场渲染。
- `firefly/desktop/companion_imprint.py`：图片能力同步、待办轮询和后台生成桥接。
- `firefly/runtime.py`：复用现有图片生成入口，只在确有必要时增加章节 CG 提示词辅助函数。

不新增 UI 框架，不引入数据库，不复制图片生成实现。

## 验证

### 单元测试

- 背景图库始终至少保留一张启用图片。
- 用户图片复制、删除和路径越界校验。
- 单图不轮换，多图按顺序轮换。
- 未出现的 CG 和空章节不进入图鉴查询结果。
- 大章完成只创建一个待生成任务。
- 图片模型不可用时不创建任务。
- 专属称号为空时显示“开拓者”，存在时显示专属称号。
- 成功结果原子写入后才进入图鉴；失败结果不进入图鉴。
- 旧版剧情进度可无损读取。

### 集成与回归

- 完成章节结局后，生成过场保持响应，后台生成不触碰 Tk 或 Qt UI 线程。
- 主程序正常聊天继续走原模型接口，Sidecar 仍只负责同行印记。
- 图片任务使用现有 10 分钟超时。
- Sidecar 重启后能够恢复待生成任务且不会重复生成成功章节。
- 深色、明亮主题文字与选中状态保持可读。

### 视觉检查

- 1280×720、1920×1080 和常见窄窗口下检查开始界面、图库、图鉴及生成过场。
- 背景始终铺满，无拉伸、空白或菜单遮挡。
- 圆角、选中框、气泡和文字上下居中保持完整。
