# 内置流萤面部参考技能设计

## 目标

将 `firefly-face-reference` 作为应用内置技能。用户发起流萤相关的直接图片生成时，无需在设置中开启技能库，也会使用项目随包提供的面部与眼睛参考图来稳定角色身份。

## 范围

- 将原技能说明、面部参考说明和参考图片放入 `firefly/assets/skills/firefly-face-reference/`。
- 仅在直接图片生成路径中检测流萤请求。
- 命中时，将内置参考图追加到图片工具的输入列表，并把紧凑的面部、眼睛、发饰身份描述追加到提示词。
- 用户上传的图片仍用于服装、姿势、场景、构图或风格；提示词明确内置图优先控制身份。
- 成功生成时在 `invoked_skills` 中增加 `firefly-face-reference`。

## 不做的事

- 不改变普通图片生成。
- 不改变现有图片工具权限或安全审核重试逻辑。
- 不要求开启通用技能库，也不改变技能设置页。
- 不增加依赖或新的配置项。

## 实现

`firefly/runtime.py` 使用一个小型路径辅助函数定位随包的参考图，并复用现有的 `is_firefly_self_image_request` 判断。该判断为真时，生成提示词加入身份锚点和角色分工说明，图片输入将内置参考图放在用户图片之前。安全审核重试也沿用相同的参考图输入。

若内置参考图片不存在，则跳过该图片而不阻断生成；身份提示词仍保留。这样安装包异常时不会造成生图功能整体失败。

## 验证

- 流萤请求：提示词包含眼睛与面部身份要求，图片工具收到内置参考图，调用记录包含技能名。
- 普通请求：提示词和图片输入保持原样，调用记录仅包含 `image_generation`。
- 现有直接生图与审核重试测试继续通过。
