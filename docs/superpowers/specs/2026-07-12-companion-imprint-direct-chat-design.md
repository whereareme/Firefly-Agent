# 同行印记直连对话设计

## 目标

保留同行印记的关系上下文、事件候选、用户确认和本地状态，同时让正常聊天固定直连原模型接口 `8317`。`8787` 继续作为同行印记本地服务，但不再代理或转发模型响应。

每轮正常聊天只调用一次模型，避免当前经由 Sidecar 转发产生的额外延迟。

## 请求链路

```text
同行印记 8787：读取已确认关系上下文
                 |
                 v
Firefly：把关系上下文加入本轮系统提示词
                 |
                 v
模型接口 8317：生成正常回复和隐藏关系事件候选
                 |
                 v
Firefly：移除隐藏标记并把候选提交给 8787
                 |
                 v
同行印记面板：用户选择“记下”或“暂不”
```

## 职责边界

### Firefly

- 正常聊天始终使用用户配置的原始 OpenAI 兼容地址，当前为 `http://127.0.0.1:8317/v1`。
- 每轮请求前从 Sidecar 获取已确认的关系阶段和事件摘要，并注入与现有网关相同的关系约束。
- 从模型回复中解析并隐藏同行印记控制标记，保证标记不会显示在聊天气泡或历史记录中。
- 将合法的事件候选提交给 Sidecar；提交失败不得阻断或清空正常回复。

### 同行印记 Sidecar

- 继续监听 `127.0.0.1:8787`，管理 `relationship.json`、待确认事件和确认面板。
- 提供只读关系上下文接口和事件候选提交接口。
- 校验事件种类、摘要长度、重复事件和待确认状态。
- 不再持有或转发模型 API 请求，也不读取或保存 API 密钥和完整聊天内容。

## 接口

新增两个仅允许本机访问的 JSON 接口：

- `GET /relationship/context`：返回经过裁剪的关系阶段和已确认事件摘要。
- `POST /relationship/proposals`：接收一个模型产生的 `memory`、`gift` 或 `anniversary` 候选。

接口沿用 Sidecar 现有状态校验和原子写入逻辑。请求体设定严格大小上限，拒绝未知字段。已有待确认事件时返回明确冲突状态，但不影响聊天回复。

## 设置行为

- 将“自动接管 OpenAI 兼容请求”改为“启用同行印记”。
- 开启后只启动 Sidecar 并启用上下文同步，不修改 OpenHarness profile 的模型地址。
- 关闭后停止 Sidecar 并停止上下文注入，不触碰用户的模型地址。
- 启动时发现旧配置仍指向 `8787`，自动恢复保存的原始地址 `8317`，防止升级后继续走旧代理链路。
- 端口、Sidecar 工程路径、配置路径、启动、停止和重启功能保留。

## 对话后的原生确认弹窗

模型在最终回复末尾产生一个合法的同行印记候选后，Firefly 先移除隐藏标记并完整显示正常回复，再显示非阻塞原生确认弹窗。弹窗不抢占回复生成过程，也不延迟回复文字出现。

- 标题固定为“同行印记”。
- 展示事件类型和模型生成的简短中文摘要。
- 只提供“记下”和“暂不”两个明确操作。
- 不展示好感数值、关系进度、内部标记或模型判断过程。
- 使用 Firefly 现有绿色主色、字体、圆角、按钮和明暗主题变量。
- 同一时间只显示一个确认弹窗；后续候选按产生顺序排队。
- “暂不”直接丢弃本地候选，不写入 Sidecar。
- “记下”在后台调用 Sidecar 的原子记录接口；成功后关闭弹窗，失败时保留弹窗并显示简短错误，允许重试或暂不。
- 未确认的临时候选不持久化，Firefly 重启后不恢复。

候选在用户点击“记下”之前不进入 Sidecar 的待确认队列，避免与 Sidecar 独立管理面板产生重复弹窗或队列冲突。

## 关系变化与后续语气

确认记录后，Sidecar 保存共同经历并按现有渐进规则更新关系阶段。下一轮聊天读取更新后的关系上下文，因此变化从下一轮回复开始生效。

- `gift` 和 `anniversary` 成为以后可以自然提起的已确认共同经历。
- `memory` 表示足以改变关系的重要互动，确认后按现有规则推进最多一个关系阶段。
- 阶段顺序保持 `相识 -> 信赖 -> 亲近 -> 确认关系`，不跳级。
- 关系上下文允许语气随阶段逐渐增加信任、主动关心、依赖和暧昧程度，但不得凭空发明经历或未确认的关系。
- 普通消息数量、连续聊天、夸奖和固定关键词不会直接推动阶段。

Sidecar 新增一个原子确认接口，接收单个已由用户确认的候选并完成校验、去重、保存和必要的阶段推进。该接口只接收事件类型与摘要，不接收完整对话。

## 容错

- Sidecar 未启动、超时或返回损坏数据时，本轮聊天直接使用 `8317`，仅跳过同行印记上下文。
- 事件提交失败时记录简短诊断，不重试模型请求，也不向用户重复回复。
- Sidecar 恢复后，后续请求自动继续读取同行印记；不需要重启聊天窗口。
- 同行印记故障不能修改模型 profile、阻塞聊天或导致空回复。

## 兼容与迁移

- 保留现有关系数据文件，不重置关系阶段和已确认事件。
- Sidecar 原 OpenAI 兼容代理端点暂时保留，避免独立工程的既有测试和手动调用立即失效，但 Firefly 不再使用它进行正常聊天。
- 旧的 `companion_imprint_takeover` 字段迁移后固定清除；原始上游地址字段只用于修复遗留配置。

## 测试

- 开启同行印记后，活动 profile 仍为 `8317`，不会切换到 `8787`。
- 正常聊天只发生一次模型调用。
- 已确认事件会出现在下一轮 `8317` 请求的关系上下文中。
- 合法控制标记被隐藏并提交到 Sidecar，非法标记仅被隐藏且不改变状态。
- 合法候选在回复完整显示后触发一个非阻塞确认弹窗，用户确认前不写入状态。
- “记下”会保存事件；重要 `memory` 最多推进一个阶段，并在下一轮上下文生效。
- “暂不”、关闭弹窗和应用退出都不会保存候选。
- 多个候选按顺序显示，不会出现重叠弹窗。
- 保存失败不会关闭弹窗或影响已经显示的聊天回复。
- Sidecar 停止、超时、返回错误或已有待确认事件时，正常回复仍完整返回。
- 从旧接管配置启动时会恢复 `8317`，且不会形成 Sidecar 自转发循环。
- 设置页不再描述“自动接管 OpenAI 兼容请求”。

## 非目标

- 不新增第二次后台模型请求。
- 不根据消息数量或固定关键词直接增加好感。
- 不显示好感分数或关系进度条。
- 不改变现有的用户确认规则。
